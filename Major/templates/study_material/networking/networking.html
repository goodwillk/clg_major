Network fundamentals

A network is a group of devices connected together, wirelessly or using cables, so that they can communicate with each other and share resources. Network standards and protocols allow a wide range of devices from different manufacturers to communicate with each other. The speed at which a network operates is important to most users and you will learn about the factors that impact a network's performance. Network security is covered separately in the security topic.

Network basics

A network is a group of devices connected together so they can communicate with each other and share resources. Networks can vary hugely in terms of size and complexity. At one end of the scale, you create a network when you create a hotspot with your mobile phone, maybe to connect your laptop when you are away from home. At the other end of the scale is the internet, an ever expanding global network connecting many billions of devices.

Why people use networks

Networks have revolutionised our ability to communicate. There are many different communication systems, for example:

  1. text (sms) message
  2. email
  3. instant messaging
  4. internet telephony

All of these system require networks. Most network communication systems allow you to attach files or links to files, which makes sharing information very easy. However, it also introduces security risks as files can be malicious and links may take you to rogue websites.
Organisations have computer networks to allow users to share resources. These can be physical resources such as printers or online resources such as files.

Shared network devices

Hardware can be expensive and it makes sense to share the cost between multiple users. Not everyone needs their own laser printer or high-capacity storage device.
When devices are shared, it is important that access is managed carefully. This is especially true of shared storage devices where, even though the device is shared, you almost certainly don't want everyone to have access to every file on the device.

Shared files

When files are put into a shared area, it is very important that you set appropriate access permissions. It is possible to restrict access at the drive, folder, or file level. You can read more about file access levels in the advanced level topic on file organisation and access.

Multi-user systems

Many organisations have systems that are accessed by multiple users. For example, in your school or college, most staff will be able to access the pupil attendance system. Many multi-user systems are database systems; database management systems allow a sophisticated range of access restrictions to be put in place to ensure that data is as secure as possible.

Central backup procedures

A network allows data to be backed up centrally so that users do not need to concern themselves with backup procedures, and the possibility of the organisation losing valuable data is reduced or even eliminated. Networks allow important systems to be mirrored so that there is always a duplicate system with a copy of the live data. This allows a system failover; when the system fails, operation will switch automatically to the backup system.

Centralised deployment of software and shared software

A network allows software and software updates to be rolled out from the centre, e.g. the IT department, which saves a lot of time compared to having to visit individual computer workstations. Important updates such as new virus detection measures and security patches can be applied immediately. Software can be downloaded from servers, removing the need to transfer it onto physical storage media. Site licences allow software to be shared across large numbers of users.
Networks also allow software to be used "in the cloud". As communication speeds improve, it is possible to use online software rather than having software installed locally on your computer system. This reduces the cost of software and also allows organisations to use less powerful workstations (saving costs) as the processing is done by the server.

Accessing a network

essing is done by the server. To access a network, you need a device with a network interface card. You will also need software that allows your device to communicate; this is usually built into your device's operating system. It is very rare today to have a device that is not already set up with the hardware and software required to allow you to join a network.
There are many different types of network and you may need to find out what information you need to access the one that you are interested in. For example, to access the internet, which is the largest network in the world, you often just need a Wi-Fi key (usually called the "Wi-Fi password") or access to the mobile phone network.
In the workplace, or at school or college, you will usually need to be set up with an account on the network. Most organisations have someone who is responsible for the network and large organisations will probably have an IT department. When an account is created, you will be given a userID and password. You will also be given access to shared resources such as shared network devices and network storage. If your role changes, you may find that your permissions also change. You may find that you have to sign a user agreement before you can access the network. This will specify the things that you must and must not do.

Managing networks

If you set up a network, there are many things to think about. For example:

  1. Will the network be wired or wireless?
  2. What topology (layout) will you use?
  3. Will you have a peer-to-peer or a client-server network?
  4. What security systems will need to be put in place?

Advantages and Disadvantages of networks

Advantages
  1. The ability to share expensive hardware devices between multiple users
  2. The ability to share files
  3. The ability to have multi-user systems
  4. The ability for the organisation to track what users are doing (this can of course be a disadvantage for the individual user)
  5. The ability to back up files centrally
  6. The ability to deploy and upgrade software centrally
Disadvantages
  1. Extra hardware costs (cabling and connection devices)
  2. The cost of having to have employ someone to look after the network (depending on the size of the network)
  3. Greater security risks
