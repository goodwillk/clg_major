Object-oriented programming

Object-oriented programming (OOP) is an approach to systems development that focuses on objects and the way that they interact. Objects surround us in the real world. Look around you. For example, I am a person, and I have a desk, a computer, and a cup of water; these are all objects. How do they interact? I sit at my desk, switch on the computer, and drink some water. The OOP method of programming (or paradigm) is different to the procedural paradigm, which many programmers start out with on their learning journey. In this topic you will learn about objects and how they can be defined and worked with in a computer system. At an advanced level, you can learn about core OOP principles such as encapsulation and polymorphism. You will also find out about inheritance and aggregation and why they are important to the design of an object-oriented program. Programmers use a diagrammatical language called UML to model objects in a program and their interactions.

Objects and attributes

Objects are part of our everyday lives. If you look around, you can probably see hundreds of different objects: your mobile phone, some books, a light switch, and many more. There are also objects that exist that you can't see as easily, e.g. a bank account, a dentist appointment, or a maths lesson. In computer science, an object is anything that exists that you need to define and work with in a computer system. Consider the example of writing a computer game for a young child. The game will allow the child to look after a pet. The child can choose from a list of common pets, e.g. a dog, a cat, a hamster, or a gerbil. You could start by defining an object for each different type of pet but, in Object-Oriented Programming (OOP), you will try to generalise as much as possible. This allows you to write code that can be shared. Shared code means there is less to test and maintain in the future. You will start by identifying the properties that are common to all pets. This will allow you to write a standard definition of a pet that can be shared by any type of pet

The way you describe objects is with a list of characteristics. These are known as attributes. When you model an object in a computer system, you must decide on the list of attributes that you need to store. Real-life objects will have many, many characteristics, but not all of them will be relevant to your system. For example, a real pet will have an age and a weight, but these may not be relevant to the game. Here is the initial list of attributes for a pet object :

Classes

Once you have identified an object and its attributes, you are ready to write a class definition. A class is a specification that defines all objects of the same type. In this case, you will write a class to define a pet. Every pet in the game will be created from the same class definition and will have the same set of attributes. However, the value of each these attributes will be specific to each pet. A classic example that is used to explain the concept of a class is a 'cookie cutter'. The video clip shows how the cutter is used to make a cookie. The cutter is a template that can be used to make as many cookies as needed. Likewise, a class definition is a template for creating as many instances of objects as required for a system.

Instantiation

As detailed previously, a class is a general definition of an object. It will be used to make one or more instances of that object using a process known as instantiation. At the point of instantiation, each instance of an object is given a unique identifier so that the specific instance can be referenced when needed. This identifier is known as a reference variable; it will hold the address of a location in memory where the object is stored.

Creating a specific pet

When the code is run, a new object (an instance of the Pet class) is instantiated. The object is assigned to a reference variable. This will hold the address of the location in memory where the object can be found. In this example, the name of the reference variable is my_pet. Like any other variable name, you should choose something meaningful in the context of the program you are writing.
